# Planejamento por etapas (arquivos segmentados)

Este diretório divide o planejamento original em arquivos menores por etapa de execução, mantendo contexto suficiente em cada documento para implementação independente.

## Ordem de leitura sugerida

1. `01-etapa-base-autenticacao-infra.md`
2. `02-etapa-dashboard-planos.md`
3. `03-etapa-board-listas.md`
4. `04-etapa-cartoes-tarefas-eventos.md`
5. `05-etapa-perfil-polimento.md`

## Arquivo de referência completa

- `00-visao-geral.md`: cópia integral do planejamento consolidado original.

## Objetivo da segmentação

- Reduzir tamanho por documento.
- Facilitar execução por sprint/etapa.
- Preservar dependências e informações transversais em cada etapa (padrão visual, UX, endpoints e critérios de aceite).
